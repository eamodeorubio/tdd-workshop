SOLID

(S for Single Responsability)

  • Each artifact solves a single problem
  • Single reason to change: the problem spec changed
  • In practice: a lot of specialized & focused objects

SOLID

(O for Open/Closed)

  • Open for extension
  • Closed for modification
  • Implement only your core responsability
  • Delegate details to collaborators

SOLID

(L for Liskov)

  • Semantic subclass
  • Honor the contract you implement
  • Don't mind about the specific implementation

SOLID

(I for Interface Segregation)

  • Interfaces must have Single Responsability
  • Each role of a collaboration must be focused
  • In practice: focused role interfaces

SOLID

(D for Dependency Inversion)

  • Not to be confused with "Dependency Injection"
  • An object must depend only on its collaborator
  • Collaboration breaks down complexity
  • Collaborators only 1 level of abstraction below
  • Collaborations must be simple
  • Never code domain objects against platform APIs
  • Never code domain objects against framework APIs
